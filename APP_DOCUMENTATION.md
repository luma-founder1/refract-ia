# Refract App Documentation

## Overview

Refract is a Vibe-to-Production pipeline that transforms AI-generated code into maintainable, production-ready software. It analyzes code generated by tools like Lovable, Bolt, and Cursor, providing automated refactoring, documentation generation, and code health improvements while preserving the original functionality.

## Core Purpose

Refract addresses the "Comprehension Debt" problem in AI-generated code by:
- Analyzing code structure and identifying issues
- Providing refactoring suggestions to improve maintainability
- Generating documentation to aid understanding
- Offering automated refactoring for Pro/Team users
- Creating clean, understandable codebases from AI-generated prototypes

## Key Features

### 1. GitHub/GitLab Integration
- Secure OAuth authentication with GitHub and GitLab
- Repository and branch selection
- Automatic analysis upon connection
- Real-time synchronization with remote repositories

### 2. Code Analysis Engine
- Detection of code duplication (DRY violations)
- Identification of modules with excessive responsibilities
- Circular dependency detection
- Dead code identification
- Code health scoring (0-100 scale)
- Complexity metrics and maintainability indicators

### 3. Refract Suggestions (Semi-automated)
- Prioritized list of improvement suggestions
- Each suggestion includes:
  - Problem description
  - Affected code sections
  - Proposed solution
  - Before/after code preview
- Individual acceptance/rejection of suggestions
- Impact and effort estimation for each suggestion

### 4. Auto-Refract (Automated - Pro/Team)
- Direct application of refactoring to repositories
- Automated Pull Request generation with documented changes
- Automatic changelog creation
- Batch processing of multiple suggestions
- Conflict resolution assistance

### 5. Documentation Generator
- Automatic README creation and updating
- Function and module-level documentation
- Architecture overview diagrams
- API documentation generation
- Code comment enhancement

### 6. Advanced Features (V2+)
- Drift Monitor: Weekly alerts for code degradation
- CI/CD Integration: Automatic analysis on each PR
- Team Dashboard: Shared visibility and collaboration
- Slack/Discord notifications
- IDE Extensions for Cursor/VSCode
- Refract Score Badge for public READMEs

## How to Use Refract

### Getting Started

1. **Authentication**
   - Visit the application URL
   - Click "Sign In" on the splash screen
   - Choose GitHub or GitLab OAuth
   - Authorize Refract to access your repositories
   - Complete any required profile information

2. **Connecting a Repository**
   - Navigate to the Repositories page
   - Click "Connect Repository"
   - Select the desired repository from your account
   - Choose the branch to analyze (default: main/master)
   - Confirm connection
   - Wait for initial analysis to complete (typically <5 minutes)

3. **Reviewing Analysis Results**
   - Go to Projects page
   - Select your connected repository
   - Review the health score (0-100)
   - Examine the breakdown of issues found
   - Navigate to the Suggestions tab for detailed recommendations

4. **Working with Suggestions**
   - Browse the prioritized list of suggestions
   - Click on any suggestion to view details
   - Review the "Before" and "After" code previews
   - Accept or reject individual suggestions
   - Add comments or notes to suggestions
   - Group related suggestions for batch processing

5. **Applying Changes**
   - **Free/Pro Users**: Manual application
     - Copy suggested changes from the preview
     - Apply changes locally or via your IDE
     - Commit and push changes manually
   - **Pro/Team Users**: Auto-Refract
     - Select multiple suggestions to apply
     - Click "Generate PR" to create a pull request
     - Review the generated PR description and changes
     - Create the PR in your repository
     - Monitor and merge as usual

6. **Generating Documentation**
   - Navigate to the Documentation tab in Project View
   - Click "Generate Documentation"
   - Select documentation types to generate (README, API docs, etc.)
   - Review generated content
   - Apply to repository manually or via PR (Pro/Team)

### Advanced Usage

#### Team Collaboration

## Technical Details

### Architecture
Refract follows a modular architecture separating concerns:
- **Frontend**: React application (Web/Electron)
- **Backend**: Node.js/TypeScript services
- **Analysis Engine**: AST-based code parsing
- **AI Layer**: Claude API for intelligent suggestions
- **Storage**: SQLite (local) / Supabase (cloud)
- **Authentication**: Supabase Auth + OAuth providers

### Data Flow
1. User connects repository via OAuth
2. Backend fetches repository contents
3. Analysis Engine parses code using AST
4. AI Layer generates suggestions based on patterns
5. Results stored and presented in UI
6. User actions trigger backend processes
7. Changes applied via Git operations (when authorized)

### Security Measures
- User code never logged or stored permanently
- API keys stored in secure storage (Electron safeStorage)
- Analysis runs in isolated child processes
- No direct access to UI from analysis processes
- Secure OAuth implementations
- Regular security audits and penetration testing

### Performance Characteristics
- Initial analysis: <5 minutes for typical repositories
- Incremental updates: <30 seconds for small changes
- Memory usage: Optimized for large codebases
- Concurrent analysis: Supports multiple repositories
- Caching: Intelligent caching of analysis results

## Glossary

- **Vibe Coding**: Rapid development using AI-assisted coding tools
- **Comprehension Debt**: The gap between code that works and code that is understood
- **Refactor**: Improving code structure without changing external behavior
- **DRY**: Don't Repeat Yourself principle
- **AST**: Abstract Syntax Tree (used for code analysis)
- **PR**: Pull Request (Git/GitHub/GitLab concept)
- **CI/CD**: Continuous Integration/Continuous Deployment
